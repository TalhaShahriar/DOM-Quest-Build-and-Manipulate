Web Development Q&A

1. What is the Document Object Model (DOM)?

The Document Object Model (DOM) is a programming interface for HTML and XML documents. It represents the structure of a document as a tree-like hierarchy of objects, where each object represents a part of the document, such as elements, attributes, and text. The DOM provides a way for programs to dynamically access and manipulate the content, structure, and style of web documents.

2. How do you select an element from the DOM?

There are several ways to select elements from the DOM in JavaScript:

  • getElementById(): Selects an element by its ID attribute
  • querySelector(): Selects the first element that matches a CSS selector
  • querySelectorAll(): Selects all elements that match a CSS selector
  • getElementsByClassName(): Selects elements by their class name
  • getElementsByTagName(): Selects elements by their tag name
3. What is event delegation in the context of the DOM, and why is it useful?

Event delegation is a technique where you attach a single event listener to a parent element instead of attaching multiple listeners to individual child elements. When an event occurs on a child element, it bubbles up through the DOM tree, and the parent element's event listener can handle it. This is useful because:

  • It reduces the number of event listeners, improving performance
  • It works for dynamically added elements without needing to attach new listeners
  • It simplifies code by centralizing event handling logic
4. How do you manipulate an element's attributes and styles using the DOM?

To manipulate an element's attributes and styles using the DOM:

  • For attributes: Use element.getAttribute(), element.setAttribute(), element.removeAttribute()
  • For styles: Use element.style.propertyName = 'value' or element.style.setProperty()
  • To add/remove classes: Use element.classList.add(), element.classList.remove(), element.classList.toggle()
